# Validation

Data validation infrastructure for the R15 experiment. The `validate.py` script
implements three checks: schema compliance, SHA-256 integrity, and completeness.

## Usage

```bash
cd experiment

# Run all checks
python validation/validate.py --all

# Or run individual checks
python validation/validate.py --check-completeness
python validation/validate.py --check-schemas
python validation/validate.py --check-integrity

# Regenerate the checksum file after intentional data changes
python validation/validate.py --regenerate-checksums
```

## Checks

### 1. Completeness check (`--check-completeness`)

Verifies that all expected files exist on disk:

- **15 L3 session JSONL files** in `L3_sessions/` (Runs 2-9)
- **13 L4 result files** in `L4_analysis/` (run5/run6/run7/run8/run9 + 3 robustness analyses + 5 run5 detail files)
- **3 root result files** at the package root (`results_v2_global.json`, `results_v3_local.json`, `results_v4_resolution.json`)

Counts total session log lines and reports them.

### 2. Schema check (`--check-schemas`)

Validates session records and aggregated results against JSON schemas in `schemas/`:

- `session_record.schema.json` â€” applies to every line of every L3 JSONL file. Samples first/middle/last + every 500th line per file.
- `aggregated_results.schema.json` â€” applies to the three root `results_v*.json` files.

Uses `jsonschema` if installed; falls back to a built-in minimal checker that enforces required keys and basic types.

### 3. Integrity check (`--check-integrity`)

Verifies SHA-256 hashes of all checksummed data files against `checksums.sha256`. The checksummed set includes:

- All 15 L3 session JSONL files
- All 13 L4 result files
- All 3 root results files
- `local_brand_specs.json`, `L3_sessions/metadata.yaml`, `L1_configuration/models.yaml`

Total: 34 files. Run `--regenerate-checksums` after intentional data changes (e.g., adding a new run).

## Schemas

`schemas/session_record.schema.json`:
JSON Schema for one line of any L3 JSONL file. Required keys: `timestamp`, `model`, `prompt_type`, `prompt`. Allowed `prompt_type` values are enumerated. All other fields are optional and typed.

`schemas/aggregated_results.schema.json`:
JSON Schema for the three root `results_v*.json` files generated by `L4_analysis/aggregate_runs_2_to_4.py`. Required keys: `run_id`, `n_calls`, `models`, `dimensional_weights`, `dci_aggregate`, `schema_version`.
