# /changelog

Generate git changelog for recent commits.

## Usage

```
/changelog [options]
```

## Arguments (optional)

- No args: Last 20 commits
- `-n N`: Last N commits
- `--since [date]`: Commits since date
- `--tag`: Since last tag

## Instructions

When invoked, Claude should run:

```bash
git log --oneline -20 --pretty=format:"%h %s (%cr)"
```

For custom count:
```bash
git log --oneline -[N] --pretty=format:"%h %s (%cr)"
```

For date range:
```bash
git log --oneline --since="[date]" --pretty=format:"%h %s (%cr)"
```

## Output Format

```
=== Changelog ===

Recent commits (last 20):

abc1234 feat: add user authentication (2 hours ago)
def5678 fix: resolve login timeout issue (5 hours ago)
ghi9012 docs: update API documentation (1 day ago)
jkl3456 refactor: extract validation logic (2 days ago)
...

Commit types:
  feat: 5
  fix: 8
  docs: 3
  refactor: 4
```

## Conventional Commits

The changelog groups by commit type prefix:

| Prefix | Meaning |
|--------|---------|
| `feat:` | New feature |
| `fix:` | Bug fix |
| `docs:` | Documentation |
| `refactor:` | Code refactoring |
| `test:` | Test changes |
| `chore:` | Maintenance |
| `engine:` | Auto-generated by engine |

## Detailed Format

For release notes, use detailed format:

```bash
git log --pretty=format:"* %s (%h) - %an" --since="2025-01-01"
```
